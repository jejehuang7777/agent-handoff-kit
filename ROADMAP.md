# Roadmap

This roadmap favors adoption proof over feature volume.

## v0.1 Starter Kit

- Publish core templates.
- Include one synthetic maintainer example.
- Validate required files and common secret patterns.
- Document the public/private boundary.

## v0.2 First-Catcher Loop

- Add adoption report and workflow request issue templates.
- Publish adoption playbook and launch copy.
- Prepare Codex for OSS application draft.
- Collect first maintainer feedback.

## v0.3 Workflow Examples

Add one example only when feedback proves the need. Candidate workflows:

- issue triage
- pull request review
- release preparation
- incident follow-up
- documentation migration

## v0.4 Stronger Validation

- Check template links.
- Check that example handoff packets name one next action.
- Check that source gates separate confirmed facts from assumptions.
- Add a sample CI workflow if maintainers ask for it.

## v1.0 Maintainer-Proven Shape

The kit reaches v1.0 when maintainers outside the original project can:

- set it up from the README
- run at least one agent handoff
- open a useful adoption report
- explain which template helped and which one got in the way
