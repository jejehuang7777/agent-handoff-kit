# Fictional Web Maintainer Example

This example is synthetic. It shows how a maintainer might use the kit while
triaging a bug in a small web project.

## Scenario

A contributor reports that the settings page loses a saved theme preference
after a refresh. One agent investigates, another agent will continue the fix.

## Files

- `START_HERE.md`: local startup path
- `AGENT_CARD.md`: scope of the maintenance lane
- `TASK_BOARD.md`: active issue state
- `SOURCE_GATE.md`: source rules for browser storage behavior
- `WRITEBACK_LOG.md`: durable lessons
- `handoffs/HANDOFF_PACKET_20260611_THEME-PERSISTENCE.md`: transfer packet

## How To Try It

Ask an agent:

```text
Use examples/fictional-web-maintainer as your handoff workspace.
Read START_HERE.md and continue TASK-0001.
```
